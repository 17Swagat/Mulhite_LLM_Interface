# 🧠 MindMap LMs - Setup Guide

A Next.js chat application with **Ollama** local LLM, **Convex** cloud database, and **Clerk** authentication.

## 🎯 Features

- ✅ Real-time chat with local Ollama LLM (deepseek-r1:1.5b)
- ✅ Multi-user support with Clerk authentication
- ✅ Cloud persistence with Convex database
- ✅ Real-time sidebar updates (no polling)
- ✅ Rename and delete conversations
- ✅ Reasoning component (thinking process visualization)
- ✅ Active chat highlighting
- ✅ Protected routes with middleware

## 📋 Prerequisites

1. **Node.js 18+** installed
2. **pnpm** package manager (`npm install -g pnpm`)
3. **Ollama** installed and running locally
   - Download: https://ollama.ai
   - Model: `ollama pull deepseek-r1:1.5b`
4. **Convex account** (free tier available)
   - Sign up: https://convex.dev
5. **Clerk account** (free tier available)
   - Sign up: https://clerk.com

## 🚀 Setup Instructions

### 1. Install Dependencies

```bash
pnpm install
```

### 2. Configure Ollama

Make sure Ollama is running:

```bash
# Start Ollama server
ollama serve

# In another terminal, pull the model
ollama pull deepseek-r1:1.5b
```

Verify it's working:

```bash
ollama list
# Should show deepseek-r1:1.5b
```

### 3. Set up Convex

```bash
# Login to Convex
npx convex login

# Initialize Convex project
npx convex dev
```

This will:
- Create a new Convex project (or link existing)
- Deploy your schema and functions
- Start watching for changes

Keep this terminal open - it syncs your Convex functions.

### 4. Set up Clerk

1. Go to https://dashboard.clerk.com
2. Create a new application
3. Copy your keys from the dashboard
4. Create `.env.local` file:

```env
# Clerk Keys
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_...
CLERK_SECRET_KEY=sk_test_...

# Convex
CONVEX_DEPLOYMENT=...  # Auto-generated by `npx convex dev`
NEXT_PUBLIC_CONVEX_URL=https://...  # Auto-generated

# Clerk URLs
NEXT_PUBLIC_CLERK_SIGN_IN_URL=/login
NEXT_PUBLIC_CLERK_SIGN_UP_URL=/login
NEXT_PUBLIC_CLERK_SIGN_IN_FALLBACK_REDIRECT_URL=/chat
NEXT_PUBLIC_CLERK_SIGN_UP_FALLBACK_REDIRECT_URL=/chat
```

5. In Clerk dashboard, add Convex integration:
   - Go to **JWT Templates**
   - Create a new template for Convex
   - Copy the Issuer URL
   - Add it to `convex/auth.config.ts` (already configured)

### 5. Run Development Server

```bash
# In one terminal: Convex dev server
npx convex dev

# In another terminal: Next.js dev server
pnpm dev
```

Open http://localhost:3000

```
src/
├── app/
│   ├── (auth)/login/        # Login page (Clerk)
│   ├── (chat)/
│   │   └── chat/
│   │       ├── page.tsx     # New chat page
│   │       └── [id]/        # Chat conversation page

│       └── persist-chat/    # Ollama streaming endpoint
├── components/
│   ├── my/
│   │   ├── AppSidebar.tsx   # Sidebar with chat list
│   │   └── SidebarItem.tsx  # Individual chat item
│   └── ui/                  # UI components
├── stores/
│   └── chatStore.ts         # Zustand state management
└── middleware.ts            # Clerk auth middleware

convex/
├── schema.ts                # Database schema
├── conversations.tsx        # Queries and mutations
└── auth.config.ts           # Clerk + Convex config
```

## 📊 Database Schema

### users
- `clerkUserId` (string, indexed)
- `email` (string)
- `name` (optional string)

### conversations
- `userId` (Id<"users">, indexed)
- `title` (optional string)
- `createdAt` (number)
- `updatedAt` (number)

### messages
- `conversationId` (Id<"conversations">, indexed)
- `role` ("user" | "assistant")
- `parts` (Array<{ type: string, text?: string }>)
- `timestamp` (number)
- `metadata` (optional object)

## 🔧 Configuration

### Change LLM Model

Edit `src/app/api/persist-chat/route.ts`:

```typescript
const result = streamText({
  model: ollama('deepseek-r1:1.5b'), // Change model here
  messages: messages as any,
});
```

Available models: https://ollama.ai/library

### Customize Theme

Edit Tailwind classes in components or update `src/app/globals.css`.

## 🐛 Troubleshooting

### Ollama not responding
```bash
# Check if Ollama is running
curl http://localhost:11434/api/tags

# Restart Ollama
ollama serve
```

### Convex connection issues
```bash
# Check Convex status
npx convex dev

# Clear Convex cache
rm -rf .convex
npx convex dev
```

### Clerk authentication errors
- Verify environment variables in `.env.local`
- Check Clerk dashboard for API key validity
- Ensure JWT template is configured for Convex

### Type errors in Convex
```bash
# Regenerate Convex types
npx convex dev
```

## 🚢 Deployment

### Deploy to Vercel

```bash
# Install Vercel CLI
npm i -g vercel

# Deploy
vercel

# Add environment variables in Vercel dashboard
# (same as .env.local)
```

### Deploy Convex to Production

```bash
npx convex deploy
```

Update `.env.production` with production Convex URL.
